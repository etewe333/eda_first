## 1. 设计概述

### 1.1 功能描述

本模块为符合 RISC-V RV32I 标准的 32 位算术逻辑单元（ALU），支持以下 6 种运算操作及 zero 零标志位输出：

| 操作 | funct3 | alu_op | 描述 |
|------|--------|--------|------|
| ADD  | 000    | 0x0    | 32位无符号加法，溢出自动截断（模 2³²） |
| SUB  | 000    | 0x8    | 32位无符号减法，通过 sub_enable(alu_op[3]) 区分 |
| SLT  | 010    | 0x2    | 有符号小于比较，a < b 时结果为 1，否则为 0 |
| XOR  | 100    | 0x4    | 32位按位异或 |
| OR   | 110    | 0x6    | 32位按位或 |
| AND  | 111    | 0x7    | 32位按位与 |

### 1.2 端口定义

| 端口名 | 方向 | 位宽 | 描述 |
|--------|------|------|------|
| clock | Input | 1 | 时钟信号（模块为纯组合逻辑，未使用） |
| reset | Input | 1 | 复位信号（模块为纯组合逻辑，未使用） |
| io_a | Input | 32 | 操作数 A |
| io_b | Input | 32 | 操作数 B |
| io_alu_op | Input | 4 | 操作码：`{sub_enable, funct3[2:0]}` |
| io_result | Output | 32 | 运算结果 |
| io_zero | Output | 1 | 零标志位：`result == 0` 时置 1 |

### 1.3 操作码编码方案

采用 4 位操作码 `alu_op = {sub_enable, funct3[2:0]}`：
- `alu_op[2:0]`：RV32I 标准 funct3 编码，直接用于操作选择
- `alu_op[3]`：对应 funct7[5]，仅在 funct3=000 时生效，用于区分 ADD/SUB

此编码方案完全兼容 RV32I 指令集中 R-type 指令的 funct3 和 funct7 字段。

### 2.2 Chisel 源码设计要点

1. **操作选择器**：使用 `MuxCase` 而非 `MuxLookup`/`switch`/`VecInit` 进行操作结果选择。此设计决策源于 firtool 优化器会对 `switch` 和 `VecInit` 索引方式进行重排序，导致操作码映射错乱。`MuxCase` 基于条件匹配，不受重排序影响。

2. **有符号比较**：SLT 操作通过 `io.a.asSInt < io.b.asSInt` 实现有符号比较，经 firtool 生成为 `$signed(io_a) < $signed(io_b)`，符合 RV32I 规范。

3. **溢出处理**：ADD/SUB 使用 Chisel 原生 `+`/`-` 运算符，32位结果自动截断（模 2³²），符合 RV32I 标准的无溢出异常行为。

4. **默认值处理**：未实现的 funct3 编码（001 SLL、011 SLTU、101 SRL/SRA）通过 `MuxCase` 默认值返回 `0.U(32.W)`，不会产生未定义行为。

### 2.3 生成的 Verilog 分析

firtool 将 `MuxCase` 优化为 `wire [7:0][31:0] _GEN` 索引查找表结构：

```systemverilog
wire [7:0][31:0] _GEN =
    {{io_a & io_b},                                    // [7] funct3=111 → AND
     {io_a | io_b},                                    // [6] funct3=110 → OR
     {32'h0},                                          // [5] funct3=101 → 未实现
     {io_a ^ io_b},                                    // [4] funct3=100 → XOR
     {32'h0},                                          // [3] funct3=011 → 未实现
     {{31'h0, $signed(io_a) < $signed(io_b)}},         // [2] funct3=010 → SLT
     {32'h0},                                          // [1] funct3=001 → 未实现
     {io_alu_op[3] ? io_a - io_b : io_a + io_b}};     // [0] funct3=000 → ADD/SUB
```

**验证结论**：Verilog 数组中第一个元素对应最高索引 `[7]`，最后一个元素对应最低索引 `[0]`。经人工逐项核对，**所有 funct3 到操作的映射完全正确**。
