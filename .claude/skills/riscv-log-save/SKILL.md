# RISC-V RTL 日志保存技能

此技能保存仿真结果和Agent日志，对应 CLAUDE.md 中自动化指令流水线的第7步。

## 技能功能

保存所有验证结果和过程日志：
- 验证仿真日志文件 `sim/logs/sim_result.log` 的完整性
- 在 `agent/logs/` 目录记录需求分析和环境探测信息
- 确保日志格式符合规范要求
- 为每一轮验证创建时间戳标记

## 执行步骤

**开始执行第7步: 保存结果和日志**

1. **准备日志环境**：
   - 确保 `agent/logs/` 目录存在
   - 检查 `sim/logs/sim_result.log` 文件是否存在
   - 创建时间戳用于日志记录

2. **记录Agent日志**：
   - 将当前需求分析记录到 `agent/logs/`
   - 记录环境探测和修复逻辑
   - 包含时间戳和当前会话信息

3. **验证仿真日志**：
   - 检查 `sim/logs/sim_result.log` 文件完整性
   - 验证日志格式是否符合规范
   - 确认测试结果统计信息

4. **生成总结报告**：
   - 汇总整个自动化流程的结果
   - 记录成功/失败的步骤
   - 提供下一步建议

**完成执行第7步: 保存结果和日志 - [成功/失败]**

## Agent日志格式

Agent日志应包含以下内容：

### 日志文件命名
- 时间戳格式：`agent/logs/session_YYYYMMDD_HHMMSS.log`
- 或使用递增编号：`agent/logs/session_001.log`

### 日志内容结构
```
=== AGENT LOG - [时间戳] ===
会话ID: [会话标识]
项目路径: [项目绝对路径]
环境指纹: [环境信息]

=== 需求分析 ===
[用户需求描述]
[功能要求分析]
[设计决策]

=== 环境探测 ===
[环境校验结果]
[工具链检查结果]
[修复措施（如有）]

=== 执行流水线 ===
步骤1: 环境指纹校验 - [状态]
步骤1: 工具检查 - [状态]
步骤2: 目录初始化 - [状态]
步骤3: 代码生成 - [状态]
步骤4: 生成RTL - [状态]
步骤4: 语法检查 - [状态]
步骤5: Testbench编写 - [状态]
步骤5: 编译仿真 - [状态]
步骤6: 执行验证 - [状态]

=== 验证结果 ===
测试用例总数: [数量]
通过数: [数量]
失败数: [数量]
仿真结果: [PASS/FAIL]

=== 总结和建议 ===
[整体评估]
[建议的下一步]
```

## 仿真日志验证标准

### 必须包含的内容
1. 所有测试用例输出（格式：`[TEST id] <OP>: a=<VAL> b=<VAL> | expect=<VAL> got=<VAL> | PASS/FAIL`）
2. 测试总结（`=== TEST SUMMARY ===`）
3. 通过/失败统计
4. 仿真程序的退出码

### 验证标准
- 日志文件必须存在且非空
- 必须包含完整的测试输出
- 必须有明确的总结部分
- 必须有仿真退出码（应为0表示成功）

## 结果判断标准

### 成功标准
1. `agent/logs/` 目录中成功创建日志文件
2. `sim/logs/sim_result.log` 验证通过（存在、格式正确、包含测试总结）
3. 所有验证步骤记录完整

### 失败标准
1. `agent/logs/` 目录创建失败
2. `sim/logs/sim_result.log` 不存在或格式不正确
3. Agent日志记录失败

## 常见问题处理

### agent/logs/ 目录不存在
**原因**: 目录未初始化
**解决方案**: 自动创建目录 `mkdir -p agent/logs/`

### 仿真日志文件不存在
**原因**: 步骤6执行失败或日志路径错误
**解决方案**: 检查步骤6的执行结果，确认仿真是否成功运行

### 日志格式不正确
**原因**: testbench 输出格式不符合规范
**解决方案**: 确保步骤5中testbench使用正确的输出格式

## 自动化集成

### 与主协调技能集成
此技能通常由 `riscv-pipeline-main` 技能在流水线最后一步调用。

### 环境依赖
- 需要步骤6成功执行并生成 `sim/logs/sim_result.log`
- 需要项目目录具有写入权限

## 注意事项

1. **时间戳**: 使用一致的日期时间格式
2. **文件权限**: 确保有足够的权限创建日志目录和文件
3. **日志轮转**: 考虑实现简单的日志轮转机制，避免单个文件过大
4. **敏感信息**: 避免在日志中记录敏感信息（如完整路径、用户名等）

- `riscv-sim-run` - 仿真运行技能（上一步，生成仿真日志）
- `riscv-pipeline-main` - 主协调技能（协调整个流水线）
